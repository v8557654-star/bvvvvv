# 🛰 Orbit — мессенджер нового поколения

Полнофункциональный мессенджер в стиле Telegram, написанный на Node.js + WebSocket + ванильный JS, с собственным AI-ассистентом.

## ✨ Возможности

### Базовые (как в Telegram)
- 🔐 Регистрация/вход с JWT-токенами и хешированием паролей (bcrypt)
- 💬 Личные чаты, **группы** и **каналы**
- ⚡ Реальное время через WebSocket (мгновенная доставка)
- 📎 Отправка файлов и изображений (до 25 МБ)
- 😊 Реакции на сообщения (24 эмодзи)
- ↩️ Ответы (reply) на сообщения
- ✏️ Редактирование и удаление своих сообщений
- 👁 Индикатор «печатает...»
- 🟢 Статусы онлайн/офлайн в реальном времени
- 🔔 Непрочитанные счётчики и звуковые уведомления
- 🔍 Поиск чатов и людей
- 📌 Закреплённые/заглушенные чаты
- 🎨 Тёмная тема в стиле Telegram
- 📱 Адаптив для мобильных
- ✨ Markdown в сообщениях (`**жирный**`, `_курсив_`, `` `код` ``, ссылки)

### Сверх Telegram
- 🤖 **Orbit AI** — встроенный чат-ассистент с командами:
  - `/help` — список команд
  - `/summary` — краткий пересказ переписки
  - `/translate en Текст` — перевод (демо)
  - `/idea стартап` — генерация идей
  - Просто пишите — отвечает на сообщения
- Создаётся автоматически при регистрации
- Архитектура готова к подключению реальных LLM (OpenAI, Claude) — точка расширения в `server/ai.js`

## 🚀 Запуск

```bash
cd orbit
npm install
npm start
```

Откройте http://localhost:3000

Создайте 2 аккаунта (можно в разных окнах/инкогнито), найдите второго через поиск и начните чат. Также сразу появится чат с **Orbit AI**.

## 🔧 Подключение настоящего AI

Откройте `server/ai.js` и замените функцию `smartReply()` на вызов любого LLM API:

```js
async function smartReply(text) {
  const r = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: { 'Authorization': 'Bearer ' + process.env.OPENAI_KEY,
               'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [{ role: 'user', content: text }]
    })
  });
  const d = await r.json();
  return d.choices[0].message.content;
}
```
(не забудьте сделать `handle` асинхронной и обработать `await` в `server/index.js`)

## 🛡 Безопасность

- Пароли хранятся в bcrypt-хешах
- JWT с секретом из ENV (`JWT_SECRET`)
- Загружаемые файлы переименовываются (защита от path traversal)
- SQL — параметризованные запросы (better-sqlite3)

Для продакшена обязательно:
- Установите `JWT_SECRET` в окружении
- Включите HTTPS (за nginx/Caddy)
- Добавьте rate-limiting
